# CV 改写规则

## 允许的改写操作

### 1. 措辞替换（✅ 允许）
用 JD 的关键词替换同义表达：
- 「开发了功能」→「设计并实现了功能」（如果 JD 强调 design）
- 「负责数据分析」→「主导数据驱动决策」（如果 JD 强调 decision-making）
- 「协作完成项目」→「跨职能团队协作交付」（如果 JD 强调 cross-functional）

### 2. 技能排序调整（✅ 允许）
将与 JD matched_skills 重叠的技能移到技能列表前面。

### 3. Summary 重写（✅ 允许）
完全重写 summary 段，但只能基于 cv_parsed.json 中已有的经历和技能。

### 4. 量化补充（✅ 允许，有条件）
只能使用用户原文中已出现的数字。不能凭空添加百分比或数量。

## 绝对禁止项

### ❌ 禁止添加新经历
不得在 experience 中添加用户没做过的工作、项目或职位。

### ❌ 禁止添加新技能
即使 JD 要求某技能而用户没有，也不得将其加入 skills 列表。

### ❌ 禁止修改关键事实
- 不得改变任职时间
- 不得改变公司名或职位名
- 不得改变学历信息

### ❌ 禁止发明数字
「将效率提升了 40%」——如果原文没有这个数字，禁止添加。

## 验证检查
改写完成后，cv-writer 必须自检：
1. experience 的条数和每条的公司/职位名是否与原文一致？
2. skills 的数量是否 ≤ 原文？
3. 有没有出现原文中没有的专有名词（公司名、项目名、技术名）？

如果任何一项检查失败，输出 `CV_WRITE_BLOCKED` 并说明原因。
